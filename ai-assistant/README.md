# AI-помощник — инструкция по запуску

Этот бот уже написан. Разбито на два этапа: сначала простой рабочий вариант
без Google (можно протестировать сегодня, всего 2 простых шага), потом —
необязательное добавление календаря и таблицы, когда будет время и настроение.

---

## ЭТАП A — минимальная версия (без Google Cloud)

В этом режиме бот отвечает на рутинные вопросы и эскалирует чувствительные темы
мастеру. Запись на приём бот будет предлагать через личное сообщение мастеру
(«свободных слотов не найдено, ответьте лично») — календарь подключим позже.

### Шаг A1. Ключ Anthropic (Claude API)

1. Перейдите на **console.anthropic.com** → зарегистрируйтесь/войдите.
2. **API Keys → Create Key** → скопируйте ключ (показывается только один раз).

### Шаг A2. MASTER_TELEGRAM_CHAT_ID

1. Найдите в Telegram бота **@userinfobot**, напишите ему `/start` —
   он пришлёт ваш числовой Telegram ID.

### Шаг A3. Заполнить `.env`

Скопируйте файл `.env.example` в новый файл с именем `.env` (в этой же папке
`ai-assistant/`). Впишите только три значения — остальное можно оставить пустым:

```
TELEGRAM_BOT_TOKEN=<токен вашего бота из BotFather>
ANTHROPIC_API_KEY=<ключ из шага A1>
MASTER_TELEGRAM_CHAT_ID=<ваш ID из шага A2>
```

### Шаг A4. Заполнить реальные данные мастера

Откройте файл `knowledge_base.py`, замените примеры (цены, адрес, часы работы,
инструкции подготовки) на реальные данные мастера-пилота.

### Шаг A5. Локальный запуск и тест

В терминале, находясь в папке `ai-assistant/`:

```
pip install -r requirements.txt
python bot.py
```

Если увидите `Бот запущен, ждёт сообщения...` — всё работает. Дальше протестируйте
с второго Telegram-аккаунта (попросите кого-то или используйте телефон):
1. Рутинный вопрос («сколько стоит чистка лица?») — бот должен ответить сам.
2. Что-то про здоровье («у меня герпес, можно ли делать чистку?») — сообщение
   должно прийти вам в личку без обработки, с пометкой "⚠️ Требуется ваш личный ответ".
3. Просьба записаться — на этом этапе бот скажет, что свободных слотов не нашёл,
   и попросит клиента подождать вашего личного ответа (календарь подключим на этапе B).

**На этом уже есть работающий, тестируемый продукт для первого пилота.**

## ЭТАП B — добавляем Google Calendar + Google Sheets (необязательно, можно позже)

Это отдельный, более технический шаг — делайте его не спеша, отдельной сессией,
когда будет время, или попросите меня разобрать его максимально подробно вместе.

### Шаг B1. Google Cloud: сервис-аккаунт

1. **console.cloud.google.com** → создать новый проект.
2. **API и сервисы → Библиотека** — включить **Google Sheets API** и **Google Calendar API**.
3. **API и сервисы → Учётные данные → Создать учётные данные → Сервисный аккаунт**.
4. Открыть сервис-аккаунт → **Ключи → Добавить ключ → Создать новый ключ → JSON**
   — скачается `service_account.json`, положите его в папку `ai-assistant/`.
5. Скопируйте email сервис-аккаунта (вида `имя@проект.iam.gserviceaccount.com`).

### Шаг B2. Расшарить таблицу и календарь

1. Google Таблица «AI-помощник — данные» → Настройки доступа → email сервис-аккаунта → **Редактор**.
2. Google Календарь → Настройки календаря → Доступ конкретным пользователям →
   тот же email → **Внесение изменений в мероприятия**.
3. Скопируйте ID таблицы (часть ссылки между `/d/` и `/edit`) и ID календаря
   (Настройки календаря → Интеграция → Идентификатор календаря).

### Шаг B3. Дополнить `.env`

```
GOOGLE_SERVICE_ACCOUNT_FILE=service_account.json
GOOGLE_SHEET_ID=<ID таблицы>
GOOGLE_CALENDAR_ID=<ID календаря>
```

Перезапустите `python bot.py` — теперь бот сам предлагает время из календаря и
логирует всё в таблицу.

## ЭТАП C — деплой на Railway (чтобы бот работал 24/7)

Можно сделать после Этапа A, не дожидаясь Этапа B.

1. Зарегистрируйтесь на **railway.app** (проще всего через GitHub-аккаунт).
2. Создайте проект, загрузите код (или подключите GitHub-репозиторий — помогу собрать вместе, если понадобится).
3. В настройках проекта добавьте переменные окружения — те же, что в `.env`
   (для Google — содержимое `service_account.json` целиком в переменную `GOOGLE_SERVICE_ACCOUNT_JSON`).
4. Команда запуска: `python bot.py`.
5. Deploy.

Если Railway окажется недоступен в вашем регионе — переключимся на Render.com,
код останется тем же.
