## 1. Что уже точно работает — копировать без раздумий

| Паттерн | Почему это работает | Кто это доказал |
|---|---|---|
| **Мессенджер как единственный интерфейс** — никакого отдельного приложения | Пользователь не устанавливает ничего нового, что снимает главный барьер входа | irel, HumanInbox, text.ai |
| **Один узкий агент с одной задачей**, а не «команда агентов» с самого начала | Полный «оркестр» агентов — это v2, не MVP; узкий агент проще сделать надёжным | Pancake, CoreNews |
| **Черновик + подтверждение человеком** на старте, а не полная автономность | Снимает тревогу пользователя перед доверием к AI; можно включить полную автономность позже, когда доверие есть | Zaturn/Androva, Cobura |
| **Эскалация к человеку при любой неуверенности** — не пытаться быть умнее, чем нужно | Встречается во всех надёжных продуктах, не только в нашей спеке — это индустриальный стандарт, а не наша особенность | irel, Scarlyfy, соло lead-mining кейсы |
| **RAG поверх прайса/инструкций мастера** как база для рутинных ответов | Это единственное, что реально нужно «знать» боту — не нужен сложный ИИ, нужен хороший источник фактов | Chatbase (RAG-ядро, $9M ARR соло) |
| **Готовый отраслевой шаблон** (пред-заполненные Q&A под нишу) вместо конструктора с нуля | Мастер тратит минуты на заполнение, а не часы на настройку логики | Solvea (шаблон именно под med-spa) |
| **`wa.me`-ссылка с предзаполненным текстом** в Instagram/на сайте | Почти бесплатный «быстрый выигрыш» ещё до готовности полного бота | WASender/ChatWith/SuperLemon, lead-gen шаблон на n8n |
| **Единый общий бот с привязкой аккаунта**, а не требование каждому пользователю заводить свой токен/API-ключ | Own-setup убивает ~20% регистраций на трении настройки | Vyneron (прямое признание автора) |
| **Не становиться платёжным оператором** — направлять оплату через уже существующие способы мастера | Избегает лицензионных/комплаенс-рисков, экономит месяцы разработки | invioTrack (сознательное архитектурное решение) |
| **Разовая оплата за настройку** как первая продажа, подписка — потом | Продать разовую услугу проще, чем уговорить на регулярный платёж с нуля | ExpenseBot ($149 DFY), соло-консультант с $23k на 4 агентах |

## 2. Типичные ошибки — сознательно избегать

| Ошибка | Что происходит | Источник урока |
|---|---|---|
| Маркировать заявку в Meta как «AI-ассистент» | Более долгая и придирчивая проверка, особенно если ниша касается здоровья/медицины | Scarlyfy — 4-6 недель ожидания; правильная маркировка: «запись на приём» |
| Строить голосовой канал/телефонию в MVP | Голос требует отдельного сложного пайплайна (STT+LLM+TTS+телефония) — это правильная «фаза 2», не старт | Dialzara, Slang.ai — оба 🔴 сложные именно из-за голоса |
| Пытаться конкурировать чистой функциональностью с зрелыми игроками (Бьюти Бот, Wahelp, Suvvy, AI Beauty Bot) | У них команды разработки и годы на рынке — соло no-code сборка не догонит по фичам | см. конкурентный анализ в переписке — дифференциация должна быть не в функциях, а в позиционировании/цене/дистрибуции |
| Собирать «полный оркестр агентов» до того, как один агент подтверждён | Множит точки отказа и сложность без валидации базовой ценности | CoreNews и Pancake — оба начинали с одного агента, не с оркестра |
| Игнорировать «конкуренцию с привычками» | Мастер может годами вести заметки вручную — одного наличия технологии недостаточно для смены поведения | Прямая цитата основателя Scarlyfy |
| Обрабатывать/хранить чувствительные данные (здоровье) внутри бота вместо немедленной эскалации | Юридический и этический риск на пустом месте, когда в этом нет необходимости | Уже заложено в нашу спеку — держать эту границу жёстко |
| Выбирать неофициальный WhatsApp API (например, Whapi.Cloud) без явного понимания риска бана номера | Быстрее в старте, но нестабильно и может отключиться без предупреждения | WhatsApp AI Bot Template — сам автор явно предупреждает об этом выборе |

## 3. Технический reference для сборки (для Claude Code иno-code)

**Стек, который уже выбран (см. no-code план):**
- Оркестрация: Make.com (n8n — запасной вариант при росте сложности)
- База: Airtable (таблицы Clients / Conversations / Bookings)
- Канал старта: Telegram Bot API (без approval)
- Каналы v1.5: WhatsApp Business Cloud API (официальный, не Whapi.Cloud — см. пункт про риск бана) + Instagram Graph API — обе требуют Meta App Review, маркировать как «appointment scheduling»
- AI-логика: Claude API — классификация (рутина/эскалация) + генерация ответа + RAG поверх прайса/инструкций мастера
- Календарь: Google Calendar API (бесплатно) + YCLIENTS/Altegio REST API (партнёрский доступ)

**Порядок сборки (validated by research, не только наша интуиция):**
1. Один канал (Telegram), один агент, одна задача (классификация + рутинный ответ) — как в CoreNews/Pancake, не оркестр сразу.
2. RAG-слой поверх прайса/адреса/инструкций мастера (ядро Chatbase) — здесь бот получает «знания», не более сложный промпт.
3. Эскалация к человеку по умолчанию при неуверенности — жёсткое правило, не опция.
4. Интеграция календаря (Google Calendar сначала — бесплатно и просто, YCLIENTS/Altegio — по запросу от мастеров, у которых уже есть).
5. Подтверждение мастером одним кликом (inline-кнопки Telegram) — режим «черновик + одобрение», не полная автономность на старте.
6. Только после стабильной Telegram-версии — параллельно поданная заявка в Meta для WhatsApp/Instagram (подана в день 1, но продукт не ждёт её одобрения).

**Что сознательно НЕ строим на этом этапе:** голосовой канал, мультиагентный оркестр, приём платежей внутри бота, собственный API-ключ для каждого мастера, DIKIDI и другие сервисы записи кроме двух выбранных.
