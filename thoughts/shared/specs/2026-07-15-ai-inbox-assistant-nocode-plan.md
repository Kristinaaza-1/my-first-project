# План реализации через no-code — AI-помощник-администратор

Опирается на спецификацию: [2026-07-15-ai-inbox-assistant-mvp.md](2026-07-15-ai-inbox-assistant-mvp.md)

## Общая логика стека

- **Make.com** (или n8n) — «мозг», связывает все сервисы между собой без кода.
- **Airtable** — база: клиенты, диалоги, записи на приём.
- **Telegram Bot API** — первый канал, без ожидания одобрения.
- **AI API (Claude)** — понимает сообщение клиента, решает: рутина или эскалация, готовит ответ.
- **Google Calendar API / YCLIENTS-Altegio API** — расписание мастера.
- **WhatsApp Business API / Instagram Graph API** — второй трек, идёт параллельно, зависит от Meta App Review.

Рекомендация по выбору Make vs n8n: для первого no-code проекта у новичка — **Make.com** проще для старта (визуальный конструктор, много готовых шаблонов под Telegram/Google Calendar), n8n мощнее, но требует больше самостоятельной настройки (или свой сервер). Начинайте с Make.

## Неделя 0 — подготовка аккаунтов (день 1-2, делается всё сразу и параллельно)

1. Зарегистрировать бота в Telegram через @BotFather — получить токен. Это займёт 5 минут.
2. Создать аккаунт Make.com (есть бесплатный тариф для теста).
3. Создать базу в Airtable с тремя таблицами:
   - **Clients**: имя, канал, контакт, история обращений (без поля для здоровья/чувствительных данных).
   - **Conversations**: сообщение, статус (bot_handled / escalated), таймстамп.
   - **Bookings**: клиент, предложенное время, статус (pending / confirmed_by_master / declined).
4. Получить доступ к Claude API (создать API-ключ на console.anthropic.com).
5. Создать проект в Google Cloud Console и включить Google Calendar API (бесплатно) — понадобится для чтения/записи слотов.
6. **Параллельно, не дожидаясь остального:** подать заявку на Meta App Review для WhatsApp Business API и Instagram messaging — это самая долгая и непредсказуемая часть, поэтому стартует в день 1, а не в конце.
7. Если у части ваших пилотных мастеров уже есть YCLIENTS/Altegio — написать в partner@yclients.tech или посмотреть на developer.alteg.io для получения партнёрского доступа к API.

## Неделя 1 — базовый сценарий на Telegram

1. В Make: сценарий «Telegram → новое сообщение» как триггер.
2. Записываем входящее сообщение в Airtable (Conversations).
3. Модуль вызова Claude API с промптом вида: «Определи, является ли сообщение рутинным вопросом (цена/адрес/подготовка/запись) или требует личного ответа мастера (здоровье, жалобы, нестандартные случаи). Если рутинное — ответь. Если нет — верни пометку ESCALATE без анализа содержания.»
4. Ветвление в Make: если ESCALATE — переслать оригинальное сообщение мастеру в отдельный Telegram-чат/канал «Входящие для меня», без обработки ботом. Если рутина — отправить сгенерированный ответ клиенту напрямую.

Важно: держите промпт эскалации консервативным — при любой двусмысленности лучше передать мастеру, чем ошибиться на теме здоровья.

## Неделя 2 — расписание и запись

1. Подключить модуль Google Calendar в Make: чтение свободных слотов.
2. Когда клиент подтверждает желание записаться — Claude формирует предложение слота на основе свободного времени.
3. При согласии клиента — создаётся запись в Airtable (Bookings, статус pending) и в Telegram-чат мастера уходит сообщение с двумя кнопками: «Подтвердить» / «Отклонить» (это и есть «одним кликом» из спеки — реализуется через Telegram inline keyboard, Make это поддерживает).
4. По нажатию «Подтвердить» — Make создаёт событие в Google Calendar (или через YCLIENTS/Altegio API, если мастер там) и обновляет статус в Airtable.

## Неделя 3 — внутреннее тестирование

1. Прогнать сценарий на себе / доверенном человеке: типичные рутинные вопросы, типичный «острый» вопрос про здоровье — проверить, что эскалация срабатывает, а не отвечает бот.
2. Проверить экспорт данных из Airtable (Кристина уже зафиксировала: клиентские данные принадлежат мастеру и должны экспортироваться в любой момент) — тут это буквально «Export CSV» из Airtable, ничего дополнительно строить не нужно.
3. Проверить, что при недоступности Claude API или сети сценарий не «зависает» без ответа клиенту (простая заглушка «мастер скоро ответит»).

## Неделя 4 — первый пилот

1. Подключить 1-2 реальных мастеров из ваших контактов на Telegram-версию.
2. Наблюдать: закрывает ли бот большинство рутинных вопросов, не путает ли эскалацию.
3. Собирать прямую цитату «буду платить» / фидбек — это ваша метрика успеха по спеке.
4. Параллельно проверить статус Meta App Review — если одобрено, начать пилот WhatsApp/Instagram аналогично.

## Риски и как на них реагировать

- **Meta отклонит заявку с первого раза** — это частая практика, не повод останавливаться: обычно нужно уточнить use case/privacy policy и подать повторно. Telegram-трек тем временем продолжает давать реальные данные с пилота.
- **Промпт для эскалации ошибается** («рутинный» вопрос на деле деликатный) — держите порог чувствительности высоким: при сомнении эскалировать, тестировать промпт на реальных примерах переписки перед запуском на живых клиентах.
- **Незнакомый no-code стек** — Make имеет собственные обучающие сценарии и шаблоны именно под связку «Telegram bot + AI + Calendar», не нужно собирать с нуля — попросите меня разобрать конкретный шаг, когда дойдёте до него в интерфейсе.
